# Design Context

## Users

**시설 관리자/운영자** — 데이터센터 관제실에서 실시간 모니터링. 이상 징후 발견 후 빠른 조치가 핵심.
**경영진/의사결정자** — 회의실이나 사무실에서 현황 브리핑. 한눈에 상태 파악이 핵심.

같은 시스템, 다른 뷰: 운영자는 상세 데이터, 경영진은 요약 대시보드.

## Brand Personality

**깔끔, 명료, 절제.**

데이터가 스스로 말하게 한다. 인터페이스는 뒤로 물러나고 정보가 전면에 선다.
불필요한 장식 없이 구조와 위계만으로 명확함을 만든다.

## Aesthetic Direction

### 테마: 다크 + 라이트 양쪽 지원

같은 컴포넌트가 다크/라이트 양쪽 테마를 가진다.

**다크 테마**: 관제실/모니터링 맥락. 밀도 높은 정보, 데이터 강조.
**라이트 테마**: 브리핑/리포트 맥락. 깨끗한 배경, 읽기 편한 구조.

### 안티 레퍼런스: AI 생성물

절대 피할 것:
- 네온 + 그라디언트 조합 (cyan-on-dark, purple-to-blue)
- 글래스모피즘 (blur, glass card, glow border)
- 그라디언트 텍스트
- 동일 카드 그리드의 반복
- 큰 아이콘 + 둥근 모서리를 모든 곳에 배치
- 모든 버튼을 primary로 만드는 것

## Design Principles

1. **데이터 우선** — 장식이 아닌 데이터가 시각적 주인공. 차트, 숫자, 상태 표시가 가장 먼저 눈에 들어와야 한다.
2. **의도적 절제** — 모든 시각 요소는 존재 이유가 있어야 한다. "있으면 좋겠다"는 추가하지 않는다.
3. **위계로 말한다** — 색상, 크기, 여백의 차이로 정보의 중요도를 전달한다. 모든 것이 같은 크기면 아무것도 중요하지 않다.
4. **상태가 보인다** — 정상/경고/에러/오프라인이 색상만으로 즉시 구분된다. 텍스트를 읽지 않아도 상태를 안다.
5. **AI 냄새 제거** — "어떤 AI가 만들었나?"가 아니라 "디자인 에이전시가 만들었나?"라는 인상을 준다.
