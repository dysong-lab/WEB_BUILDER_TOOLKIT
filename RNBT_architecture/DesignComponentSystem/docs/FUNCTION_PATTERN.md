# 기능(Function) 패턴 — 순수 함수로 검증하는 Mixin 분류의 합리성

---

## 이 문서의 목적

기능이란 개발적 관점에서 function이다. 프로그래밍에서 이미 존재하는 순수 함수들의 목적/기능을 명세했을 때, Mixin의 목적/기능 구분과 동일한 패턴이 나타난다. 이 문서는 그 비교를 통해 Mixin 분류의 합리성을 보여준다.

> **주의:** 순수 함수와 Mixin은 **분류 패턴**이 같다는 것이지, 구현 성격이 같다는 것은 아니다. 순수 함수는 stateless이고, Mixin은 인스턴스에 상태와 메서드를 주입한다. 비교 대상은 "어떻게 구분하는가"이다.

## 순수 함수의 목적/기능 명세

### 배열 변환

| 함수 | 목적 | 기능 |
|------|------|------|
| map | 데이터를 변환한다 | 각 항목에 함수를 적용하여 새 배열을 반환한다 |
| flatMap | 데이터를 변환한다 | 각 항목에 함수를 적용하고 결과를 한 단계 펼쳐 새 배열을 반환한다 |
| flat | 중첩을 해소한다 | 중첩 배열을 지정 깊이만큼 펼친다 |

### 배열 필터링

| 함수 | 목적 | 기능 |
|------|------|------|
| filter | 데이터를 걸러낸다 | 각 항목에 조건을 적용하여 통과한 항목만 반환한다 |
| find | 항목을 찾는다 | 조건을 만족하는 첫 번째 항목을 반환한다 |
| includes | 포함을 확인한다 | 특정 값이 배열에 존재하면 true를 반환한다 |

### 배열 축약/검사

| 함수 | 목적 | 기능 |
|------|------|------|
| reduce | 데이터를 축약한다 | 각 항목을 누적 함수로 하나의 값으로 합친다 |
| some | 조건을 검사한다 | 하나라도 조건을 만족하면 true를 반환한다 |
| every | 조건을 검사한다 | 모두 조건을 만족해야 true를 반환한다 |

### 배열 구조 변경

| 함수 | 목적 | 기능 |
|------|------|------|
| sort | 순서를 정한다 | 비교 함수에 따라 항목을 정렬한다 |
| slice | 일부를 추출한다 | 시작/끝 인덱스로 구간을 잘라 새 배열을 반환한다 |
| concat | 합친다 | 두 배열을 이어 붙여 새 배열을 반환한다 |

### 함수 유틸리티

| 함수 | 목적 | 기능 |
|------|------|------|
| debounce | 호출을 제어한다 | 마지막 호출 후 일정 시간이 지나야 실행한다 |
| throttle | 호출을 제어한다 | 일정 시간 간격으로 최대 한 번만 실행한다 |
| memoize | 계산 결과를 재사용한다 | 같은 입력에 대해 이전 결과를 캐시하여 반환한다 |
| compose | 함수를 조합한다 | 오른쪽에서 왼쪽으로 함수를 순차 적용한다 |
| pipe | 함수를 조합한다 | 왼쪽에서 오른쪽으로 함수를 순차 적용한다 |

### 데이터 유틸리티

| 함수 | 목적 | 기능 |
|------|------|------|
| groupBy | 분류한다 | 키 함수의 반환값에 따라 항목을 그룹으로 묶는다 |
| uniq | 중복을 제거한다 | 동일한 값을 하나만 남기고 제거한다 |
| chunk | 나눈다 | 배열을 지정 크기의 묶음으로 분할한다 |
| merge | 합친다 | 여러 객체의 속성을 하나로 병합한다 |

## 같은 패턴: 순수 함수 ↔ Mixin

### 같은 목적, 다른 기능

순수 함수:

| 함수 | 목적 | 기능 |
|------|------|------|
| debounce | 호출을 제어한다 | 마지막 호출 후 일정 시간이 지나야 실행한다 |
| throttle | 호출을 제어한다 | 일정 시간 간격으로 최대 한 번만 실행한다 |

Mixin:

| Mixin | 목적 | 기능 |
|-------|------|------|
| FieldRenderMixin | 데이터를 보여준다 | 데이터 객체의 각 필드를 화면 요소에 1:1로 표시한다 |
| ListRenderMixin | 데이터를 보여준다 | 배열 데이터의 각 항목을 반복 생성하여 표시한다 |

구분 방식이 동일하다:
- debounce와 throttle은 같은 목적("호출을 제어한다")이지만 기능이 다르므로 별도 함수다.
- FieldRender와 ListRender는 같은 목적("데이터를 보여준다")이지만 기능이 다르므로 별도 Mixin이다.

### 기능이 같으면 같은 것

순수 함수:
- "각 항목에 함수를 적용하여 새 배열을 반환한다"는 map 하나다. 이 기능을 하는 함수가 둘일 이유가 없다.

Mixin:
- "데이터 객체의 각 필드를 화면 요소에 1:1로 표시한다"는 FieldRenderMixin 하나다. 이 기능을 하는 Mixin이 둘일 이유가 없다.

### 같은 목적 아래에서의 다수 기능 비교

순수 함수:

| 함수 | 목적 | 기능 |
|------|------|------|
| some | 조건을 검사한다 | 하나라도 조건을 만족하면 true를 반환한다 |
| every | 조건을 검사한다 | 모두 조건을 만족해야 true를 반환한다 |

Mixin:

| Mixin | 목적 | 기능 |
|-------|------|------|
| FieldRenderMixin | 데이터를 보여준다 | 기존 DOM 요소의 textContent에 필드값을 넣는다 |
| ListRenderMixin | 데이터를 보여준다 | template을 복제하여 반복 생성한다 |
| MeshStateMixin | 데이터를 보여준다 | 3D 메시의 색상을 데이터 상태에 따라 변경한다 |

some/every가 같은 목적("조건을 검사한다") 아래에서 수단으로 구분되듯, FieldRender/ListRender/MeshState도 같은 목적("데이터를 보여준다") 아래에서 수단으로 구분된다. 대상이 DOM이든 3D 메시든 목적은 동일하다.

## 결론

function을 구분하는 기준은 기능(목적 + 수단)이다. 이것은 순수 함수에서도, Mixin에서도 동일하게 적용된다.

- **목적이 같고 기능이 다르면** → 별도로 존재한다 (map ≠ filter, FieldRender ≠ ListRender)
- **기능이 같으면** → 하나로 충분하다 (map은 하나, FieldRender는 하나)
- **목적이 다르면** → 당연히 별도다 (map ≠ debounce, FieldRender ≠ CameraFocus)

Mixin을 나누는 기준은 새로운 것이 아니라, 프로그래밍에서 함수를 나누는 기준과 같다.
