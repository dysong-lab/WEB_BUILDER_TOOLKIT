# create-html-css 공통 원칙

## 색상

- 색상은 장식보다 의미와 역할 중심으로 사용한다.
- Light/Dark 전환은 hue 변경보다 명도 전환을 우선한다.
- 상태색은 의미 전달용이므로 가능한 한 테마와 독립적으로 유지한다.

## 타이포

- 시각적 계층은 크기와 여백으로 우선 만든다.
- 과도한 두께 상승으로만 계층을 만들지 않는다.

## 레이아웃

- 기본은 Flexbox
- Grid는 카드군, 대시보드 패널 등 명확한 경우만 사용
- absolute 배치는 예외적으로만 사용

## 상태와 상호작용

- hover, active, disabled는 시각적으로 구분되되 공격적으로 튀지 않게 설계
- Dark Operational만 상태성 모션을 상대적으로 허용

## 컴포넌트 생산 연결

- 출력물은 이후 `create-2d-component`가 그대로 소비할 수 있어야 한다.
- HTML은 선택자 계약을 유지하는 구조로 작성한다.
